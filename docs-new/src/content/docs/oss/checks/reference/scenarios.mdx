## `Scenario`

**Module:** `giskard.checks.core.scenario`

The recommended entry point for creating test scenarios. Chain `.interact()`, `.check()`, and related methods: each call updates and returns the same instance. Call `.run()` to execute against the SUT. You can also use `Scenario.from_sequence(...)` to assemble steps from existing specs and checks, or pass the scenario to `Suite.append()`.

<MethodCard name="Scenario()" returnType="Scenario">
  Create a new scenario.

  <Property name="name" type="str | None" defaultValue="None">Scenario name for identification.</Property>
  <Property name="trace_type" type="type[TraceType] | None" defaultValue="None">Optional custom trace type for advanced use cases.</Property>
</MethodCard>

<MethodCard name="from_sequence()" returnType="Scenario">
  Class method: build a scenario from a flat sequence of `InteractionSpec` and/or `Check` objects in execution order (alternative to chaining `.interact()` / `.check()`).

  <Property name="*components" type="InteractionSpec | Check" required>Steps to run in order.</Property>
  <Property name="name" type="str" defaultValue="'Unnamed Scenario'">Scenario name.</Property>
  <Property name="trace_type" type="type[TraceType] | None" defaultValue="None">Optional custom trace type.</Property>
  <Property name="annotations" type="dict | None" defaultValue="None">Optional annotations dict.</Property>
  <Property name="target" type="Callable | None" defaultValue="(default SUT)">Optional SUT for this scenario; if omitted, the process default target is used.</Property>
</MethodCard>

<MethodCard name=".interact()" returnType="self">
  Add an interaction to the scenario. Returns self for chaining.

  <Property name="inputs" type="value | Callable" required>Static value or callable `(trace) -> value`.</Property>
  <Property name="outputs" type="value | Callable" required>Static value, callable `(inputs) -> value`, or `(trace, inputs) -> value`.</Property>
  <Property name="metadata" type="dict | None">Optional metadata dictionary.</Property>
</MethodCard>

<MethodCard name=".check()" returnType="self">
  Add a validation check to the scenario. Returns self for chaining.

  <Property name="check" type="Check" required>A Check instance to validate the trace.</Property>
</MethodCard>

<MethodCard name=".add_interaction()" returnType="self">
  Add a pre-constructed `Interaction` or `InteractionSpec` object.

  <Property name="interaction" type="Interaction | InteractionSpec" required>The interaction or spec to add.</Property>
</MethodCard>

<MethodCard name=".extend()" returnType="self">
  Append one or more interaction specs and/or checks. Returns self for chaining.

  <Property name="*components" type="InteractionSpec | Check" required>Components to append in order.</Property>
</MethodCard>

<MethodCard name=".run()" returnType="ScenarioResult">
  Execute the scenario against the SUT and return results.

  <Property name="return_exception" type="bool" defaultValue="False">If True, return results even when exceptions occur instead of raising.</Property>
</MethodCard>

### Multi-step example

```python
from giskard.checks import Scenario, FnCheck, Equals

result = await (
    Scenario("customer_support")
    .interact(
        inputs="I need help with my account",
        outputs="I'd be happy to help! What's your account number?",
    )
    .check(FnCheck(fn=lambda trace: "help" in trace.last.outputs.lower(), name="helpful"))
    .interact(inputs="12345", outputs="Thank you! I've found your account.")
    .check(Equals(expected_value=True, key="trace.last.metadata.account_found"))
    .run()
)
```

### Dynamic interactions

```python
# Use callables for dynamic generation
def generate_response(inputs):
    if "weather" in inputs:
        return "It's sunny today!"
    return "I don't understand."

scenario = (
    Scenario("dynamic_test")
    .interact(inputs="What's the weather?", outputs=generate_response)
)
```

### Context-aware interactions

```python
scenario = (
    Scenario("context_test")
    .interact(inputs="Hello", outputs="Hi! I'm Alice.")
    .interact(
        inputs=lambda trace: f"Nice to meet you, {trace.last.outputs.split()[-1][:-1]}!",
        outputs="Nice to meet you too!",
    )
)
```

## `ScenarioResult`

**Module:** `giskard.checks.core.result`

Result of scenario execution with trace and check results.

<TypeTable name="ScenarioResult">
  <Property name="status" type="ScenarioStatus">Overall status (PASS/FAIL/ERROR/SKIP).</Property>
  <Property name="steps" type="list[TestCaseResult]">Results for each step (interact + check).</Property>
  <Property name="final_trace" type="Trace">Complete trace of all interactions.</Property>
  <Property name="passed" type="bool">True when all steps passed.</Property>
  <Property name="duration_ms" type="int">Total execution time in milliseconds.</Property>
</TypeTable>

```python
result = await test_scenario.run()

if result.passed:
    print("All checks passed!")

print(f"Total interactions: {len(result.final_trace.interactions)}")

for i, check_result in enumerate(r for step in result.steps for r in step.results):
    print(f"Check {i}: {check_result.status}")
```

## `Suite`

**Module:** `giskard.checks.scenarios.suite`

Group multiple scenarios and run them together.

<MethodCard name="Suite()" returnType="Suite">
  <Property name="name" type="str" required>Suite identifier.</Property>
  <Property name="target" type="Callable">Optional suite-level target SUT.</Property>
</MethodCard>

<MethodCard name=".append()" returnType="None">
  Add a scenario to the suite.

  <Property name="scenario" type="Scenario" required>The scenario to add.</Property>
</MethodCard>

<MethodCard name=".run()" returnType="SuiteResult">
  Run all scenarios serially.

  <Property name="target" type="Callable">Override target for this run.</Property>
  <Property name="return_exception" type="bool" defaultValue="False">Return results on exceptions.</Property>
</MethodCard>

```python
from giskard.checks import Suite, Scenario

suite = Suite(name="my_suite", target=my_sut)
suite.append(scenario1)
suite.append(scenario2)
result = await suite.run()
print(result.pass_rate)
```

## `SuiteResult`

**Module:** `giskard.checks.core.result`

Aggregate result from suite execution.

<TypeTable name="SuiteResult">
  <Property name="results" type="list[ScenarioResult]">Scenario results in order.</Property>
  <Property name="pass_rate" type="float">Fraction of non-skipped scenarios that passed.</Property>
  <Property name="duration_ms" type="int">Total execution time in milliseconds.</Property>
  <Property name="passed_count" type="int">Number of passed scenarios.</Property>
  <Property name="failed_count" type="int">Number of failed scenarios.</Property>
</TypeTable>
