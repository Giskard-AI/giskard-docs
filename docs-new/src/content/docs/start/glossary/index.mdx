This glossary defines key terms and concepts used throughout the Giskard documentation. Understanding these terms will help you navigate the documentation and use Giskard effectively.

The glossary is organized into several key areas: core concepts that form the foundation of AI testing, testing and evaluation methodologies, security vulnerabilities that can compromise AI systems, business failures that affect operational effectiveness, and essential concepts for access control, integration, and compliance.

## Core concepts

<CardGrid>
  <LinkCard title="Project" href="/hub/ui/setup/projects" description="A container for organizing related models, datasets, checks, and evaluations within Giskard Hub." />
  <LinkCard title="Model" href="/hub/ui" description="A trained machine learning model, particularly Large Language Models (LLMs) that process and generate text." />
  <LinkCard title="Agent" href="/hub/ui/setup/agents" description="An AI system or agent that can perform tasks autonomously, often using tools and following specific instructions." />
  <LinkCard title="Tool" href="/hub/ui/setup/agents" description="A function or capability that an agent can use to perform tasks, often provided by external services or APIs." />
  <LinkCard title="Dataset" href="/hub/ui/datasets" description="A collection of test cases, examples, or data points used to evaluate model performance and behavior." />
  <LinkCard title="Test Case" href="/hub/ui/datasets/manual" description="A specific input-output pair or scenario used to evaluate model behavior and performance." />
  <LinkCard title="Check" href="/hub/ui/annotate/overview" description="A specific test or validation rule that evaluates a particular aspect of model behavior (e.g., correctness, security, fairness)." />
  <LinkCard title="Evaluation" href="/hub/ui/evaluations" description="The process of testing a model against a dataset to assess its performance, safety, and compliance." />
</CardGrid>

## Access and permissions

<CardGrid>
  <LinkCard title="Access Rights" href="/hub/ui/access-rights" description="Permissions that control what users can see and do within the Giskard Hub platform." />
  <LinkCard title="Role-Based Access Control (RBAC)" href="/hub/ui/access-rights" description="A security model that assigns permissions based on user roles rather than individual user accounts." />
  <LinkCard title="Scope" href="/hub/ui/access-rights" description="The level of access a user has, which can be global (platform-wide) or limited to specific projects or resources." />
  <LinkCard title="Permission" href="/hub/ui/access-rights" description="A specific action or operation that a user is allowed to perform, such as creating projects or running evaluations." />
