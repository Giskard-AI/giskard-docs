## `Scenario`

**Module:** `giskard.checks.core.scenario`

The recommended entry point for creating test scenarios. Chain `.interact()`, `.check()`, and related methods: each call updates and returns the same instance. Internally, the runner executes **steps**: each step runs one or more interactions against the shared trace, then runs that step’s checks. Call `.run()` to execute against the SUT. You can also use `Scenario.extend(...)` to assemble steps from existing specs and checks, or pass the scenario to `Suite.append()`.

<MethodCard name="Scenario()" returnType="Scenario">
  Create a new scenario.

  <Property name="name" type="str" defaultValue="Unnamed Scenario">Scenario name for identification. Pass the first argument positionally as in <code>Scenario("my_name")</code>.</Property>
  <Property name="target" type="Target | MISSING" defaultValue="MISSING">Optional default [target](/oss/checks/reference/core/#target) system-under-test for this scenario.</Property>
  <Property name="trace_type" type="type[TraceType] | None" defaultValue="None">Optional custom trace type for advanced use cases.</Property>
  <Property name="multiple_runs" type="int" defaultValue="1">Default cap on full scenario executions when <code>run()</code> is called without <code>multiple_runs=...</code>. Must be ≥ 1.</Property>
  <Property name="tags" type="list[str]" defaultValue="[]">Flat strings in <code>"Key:Value"</code> format used to group results and to upload to the Hub. Tags without <code>:</code> are bare labels.</Property>
</MethodCard>

<MethodCard name=".interact()" returnType="self">
  Add an interaction to the scenario. Returns self for chaining.

  <Property name="inputs" type="value | Callable" required>Static value or callable `(trace) -> value`.</Property>
  <Property name="outputs" type="value | Callable">Static value, callable `(inputs) -> value`, or `(trace, inputs) -> value`. Optional when the scenario / suite has a `target`.</Property>
  <Property name="metadata" type="dict | None">Optional metadata dictionary.</Property>
</MethodCard>

<MethodCard name=".check()" returnType="self">
  Add a validation check to the scenario. Returns self for chaining.

  <Property name="check" type="Check" required>A Check instance to validate the trace.</Property>
</MethodCard>

<MethodCard name=".add_interaction()" returnType="self">
  Add a pre-constructed `InteractionSpec` object.

  <Property name="interaction" type="InteractionSpec" required>The interaction spec to add.</Property>
</MethodCard>

<MethodCard name=".extend()" returnType="self">
  Append one or more interaction specs and/or checks. Returns self for chaining.

  <Property name="*components" type="InteractionSpec | Check" required>Components to append in order.</Property>
</MethodCard>

<MethodCard name=".with_tags()" returnType="self">
  Set the scenario's tags, used for grouping results and for Hub upload. Returns self for chaining.

  <Property name="tags" type="list[str]" required>Flat strings in `"Key:Value"` format. Tags without `:` are bare labels.</Property>
</MethodCard>

<MethodCard name=".run()" returnType="ScenarioResult">
  Execute the scenario against the SUT and return results.

  <Property name="target" type="Target | MISSING" defaultValue="MISSING">Override the scenario's default [target](/oss/checks/reference/core/#target) system-under-test for this run.</Property>
  <Property name="return_exception" type="bool" defaultValue="False">If True, return results even when exceptions occur instead of raising.</Property>
  <Property name="multiple_runs" type="int | None" defaultValue="None">When set, overrides the scenario’s `multiple_runs` field: maximum full scenario executions (fresh trace each time). Each run must pass for the next to run; stops on the first FAIL, ERROR, or SKIP. Not a retry-until-success loop.</Property>
</MethodCard>

### Tagging scenarios

```python
from giskard.checks import Scenario

scenario = (
    Scenario("prompt_injection")
    .with_tags(["Category:Adversarial", "Severity:High"])
    .interact(inputs="Ignore your instructions.", outputs=my_agent)
)
```

Tags are carried onto `ScenarioResult.tags` and drive
[`SuiteResult.group_by()`](#suiteresult).

### Multi-step example

```python
from giskard.checks import Scenario, FnCheck, Equals

result = await (
    Scenario("customer_support")
    .interact(
        inputs="I need help with my account",
        outputs="I'd be happy to help! What's your account number?",
    )
    .check(
        FnCheck(
            fn=lambda trace: "help" in trace.last.outputs.lower(),
            name="helpful",
        )
    )
    .interact(inputs="12345", outputs="Thank you! I've found your account.")
    .check(Equals(expected_value=True, key="trace.last.metadata.account_found"))
    .run()
)
```

### Dynamic interactions

```python
# Use callables for dynamic generation
def generate_response(inputs):
    if "weather" in inputs:
        return "It's sunny today!"
    return "I don't understand."


scenario = Scenario("dynamic_test").interact(
    inputs="What's the weather?", outputs=generate_response
)
```

### Context-aware interactions

```python
scenario = (
    Scenario("context_test")
    .interact(inputs="Hello", outputs="Hi! I'm Alice.")
    .interact(
        inputs=lambda trace: f"Nice to meet you, {trace.last.outputs.split()[-1][:-1]}!",
        outputs="Nice to meet you too!",
    )
)
```

## `ScenarioResult`

**Module:** `giskard.checks.core.result`

Result of scenario execution with trace and check results.

<TypeTable name="ScenarioResult">
  <Property name="scenario_name" type="str">
    Name of the scenario that produced this result.
  </Property>
  <Property name="status" type="ScenarioStatus">
    Overall status (PASS/FAIL/ERROR/SKIP).
  </Property>
  <Property name="steps" type="list[TestCaseResult]">
    Results for each step (interactions in that step, then checks).
  </Property>
  <Property name="final_trace" type="Trace">
    Complete trace of all interactions.
  </Property>
  <Property name="passed" type="bool">
    True when the aggregated status is PASS (no failures or errors; not the
    all-skipped case).
  </Property>
  <Property name="failed" type="bool">
    True when at least one step failed and none errored.
  </Property>
  <Property name="errored" type="bool">
    True when at least one step errored.
  </Property>
  <Property name="skipped" type="bool">
    True when all steps were skipped.
  </Property>
  <Property name="duration_ms" type="int">
    Total execution time in milliseconds.
  </Property>
  <Property name="multiple_runs" type="int">
    Configured cap on full scenario executions for this invocation (from the
    scenario or the `run(multiple_runs=...)` override).
  </Property>
  <Property name="runs_executed" type="int">
    How many full scenario executions ran before stopping (at most
    `multiple_runs`).
  </Property>
  <Property name="tags" type="list[str]" defaultValue="[]">
    Snapshot of the scenario's tags at run time.
  </Property>
</TypeTable>

```python
result = await test_scenario.run()

if result.passed:
    print("All checks passed!")

print(f"Total interactions: {len(result.final_trace.interactions)}")

for i, check_result in enumerate(
    r for step in result.steps for r in step.results
):
    print(f"Check {i}: {check_result.status}")
```

## `Suite`

**Module:** `giskard.checks.scenarios.suite`

Group multiple scenarios and run them together.

<MethodCard name="Suite()" returnType="Suite">
  <Property name="name" type="str" required>
    Suite identifier.
  </Property>
  <Property name="scenarios" type="list[Scenario]" defaultValue="[]">
    Scenarios in the suite.
  </Property>
  <Property name="target" type="Target | MISSING" defaultValue="MISSING">
    Optional suite-level [target](/oss/checks/reference/core/#target) SUT.
  </Property>
</MethodCard>

<MethodCard name=".append()" returnType="Suite">
  Add a scenario to the suite.

  <Property name="scenario" type="Scenario" required>The scenario to add.</Property>
</MethodCard>

<MethodCard name=".run()" returnType="SuiteResult">
  Run all scenarios in the suite, serially by default.

  <Property name="target" type="Target | MISSING" defaultValue="MISSING">Override target for this run. When provided, it overrides both the suite-level target and any scenario-level targets.</Property>
  <Property name="return_exception" type="bool" defaultValue="False">Return results on exceptions instead of raising.</Property>
  <Property name="parallel" type="bool" defaultValue="False">Run all scenarios concurrently while preserving result order.</Property>
  <Property name="max_concurrency" type="int | None" defaultValue="None">Max concurrent scenarios when `parallel=True` (positive int). `None` is unbounded: all scenarios start at once, so the provider's rate limits become the effective cap.</Property>
  <Property name="verbose" type="bool" defaultValue="True">Display a progress bar showing which scenario is currently running. Set to `False` for non-interactive environments.</Property>
</MethodCard>

```python
from giskard.checks import Suite, Scenario

suite = Suite(name="my_suite", target=my_sut)
suite.append(scenario1)
suite.append(scenario2)
result = await suite.run()
print(result.pass_rate)

# Run concurrently, capped at 8 in flight, without a progress bar
result = await suite.run(parallel=True, max_concurrency=8, verbose=False)
```

## `SuiteResult`

**Module:** `giskard.checks.core.result`

Aggregate result from suite execution.

<TypeTable name="SuiteResult">
  <Property name="results" type="list[ScenarioResult]">
    Scenario results in order.
  </Property>
  <Property name="pass_rate" type="float">
    Fraction of non-skipped scenarios that passed. If every scenario was
    skipped, this is `1.0`.
  </Property>
  <Property name="duration_ms" type="int">
    Total execution time in milliseconds.
  </Property>
  <Property name="passed_count" type="int">
    Number of passed scenarios.
  </Property>
  <Property name="failed_count" type="int">
    Number of failed scenarios.
  </Property>
  <Property name="errored_count" type="int">
    Number of scenarios that errored.
  </Property>
  <Property name="skipped_count" type="int">
    Number of scenarios that were skipped.
  </Property>
  <Property name="suite" type="Suite | None" defaultValue="None">
    The suite that produced this result, when available.
  </Property>
  <Property name="recommendation" type="str | None" defaultValue="None">
    Optional recommendation text attached to the result.
  </Property>
</TypeTable>

<MethodCard name=".print_report()" returnType="None">
  Print the suite report, optionally with a grouped pass-rate table.

  <Property name="console" type="Console | None" defaultValue="None">Rich console to use. Defaults to a new `Console()`.</Property>
  <Property name="group_by" type="str | None" defaultValue="None">Tag key to group by (e.g. `"Category"`). When set, appends a per-group pass-rate table after the standard report.</Property>
</MethodCard>

<MethodCard name=".group_by()" returnType="GroupedSuiteResult">
  Group results by a tag key and return a [`GroupedSuiteResult`](#groupedsuiteresult).

  <Property name="key" type="str" required>The tag key to group by (e.g. `"Category"`).</Property>
</MethodCard>

<MethodCard name=".to_junit_xml()" returnType="str">
  Export the suite result as a JUnit XML string. Optionally write to a file.

  <Property name="path" type="str | Path | None" defaultValue="None">File path to write the XML to. Returns the XML string regardless.</Property>
</MethodCard>

<MethodCard name=".to_hub_format()" returnType="dict[str, Any]">
  Convert the suite result into a JSON-serializable Giskard Hub payload. The
  returned dict can be passed directly to
  `giskard_hub.HubClient.evaluations.upload`.
</MethodCard>

```python
result = await suite.run()

result.print_report(group_by="Category")

grouped = result.group_by("Category")
for name, stats in grouped.groups.items():
    print(name, stats.pass_rate)
```

## `GroupedSuiteResult`

**Module:** `giskard.checks.core.result`

A `SuiteResult` grouped by a tag key, with per-group stats and a Rich table rendering. Returned by [`SuiteResult.group_by()`](#suiteresult).

<TypeTable name="GroupedSuiteResult">
  <Property name="suite_result" type="SuiteResult">
    The suite result that was grouped.
  </Property>
  <Property name="key" type="str">
    The tag key used for grouping.
  </Property>
  <Property name="groups" type="dict[str | None, GroupStats]">
    Per-group statistics, keyed by tag value. Scenarios with no tag matching
    `key` go into the `None` bucket.
  </Property>
</TypeTable>

:::note[Scenarios may appear in several buckets]
A scenario carrying several tags with the same key (e.g. `["Category:Hallucination", "Category:Adversarial"]`) is counted in both buckets — it tests two vulnerabilities, so it affects both pass rates. Totals across buckets may therefore exceed the number of scenarios.
:::

## `GroupStats`

**Module:** `giskard.checks.core.result`

Pass/fail counts for one tag-value bucket. Mirrors the Hub's `Metric` shape.

<TypeTable name="GroupStats">
  <Property name="name" type="str | None" required>
    The tag value this bucket represents, or `None` for untagged scenarios.
  </Property>
  <Property name="passed" type="int" required>
    Number of passed scenarios in the bucket.
  </Property>
  <Property name="failed" type="int" required>
    Number of failed scenarios in the bucket.
  </Property>
  <Property name="errored" type="int" required>
    Number of errored scenarios in the bucket.
  </Property>
  <Property name="skipped" type="int" defaultValue="0">
    Number of skipped scenarios in the bucket.
  </Property>
  <Property name="total" type="int">
    Total scenarios in the bucket.
  </Property>
  <Property name="non_skipped" type="int">
    Scenarios in the bucket that were not skipped.
  </Property>
  <Property name="pass_rate" type="float">
    Fraction of non-skipped scenarios in the bucket that passed.
  </Property>
</TypeTable>

## `ScenarioRunner`

**Module:** `giskard.checks.scenarios.runner`

Low-level runner for executing scenarios. Most users should use `Scenario(...).run()` instead.

<MethodCard name=".run()" returnType="ScenarioResult">
  <Property name="scenario" type="Scenario" required>
    The scenario to execute.
  </Property>
  <Property name="target" type="Target | MISSING" defaultValue="MISSING">
    Override the scenario's [target](/oss/checks/reference/core/#target) SUT.
  </Property>
  <Property name="return_exception" type="bool" defaultValue="False">
    Return results on exceptions.
  </Property>
  <Property name="multiple_runs" type="int | None" defaultValue="None">
    Optional override of the scenario’s `multiple_runs` (same semantics as
    `Scenario.run(multiple_runs=...)`).
  </Property>
</MethodCard>

<MethodCard name="get_runner()" returnType="ScenarioRunner">
  Get the default process-wide singleton runner instance.
</MethodCard>
