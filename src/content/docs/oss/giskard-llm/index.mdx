**giskard-llm** is a small Python library for calling models from several vendors—OpenAI, Google Gemini, Anthropic, Azure, and others—from one place. You choose a model and the library routes the request to the correct official SDK. Other Giskard open-source tools, such as [giskard-agents](https://github.com/Giskard-AI/giskard-oss), rely on this layer so wiring and retries stay consistent across providers.

Use the **async helpers** (`acompletion`, `aembedding`, `aresponse`) with credentials from your environment, or an **`LLMClient`** when you need named setups—for example staging versus production or several API keys.

:::caution
The package ships as **pre-release** on PyPI alongside Giskard v3. Prefer pinning versions in production (`giskard-llm` follows semantic versioning intent; see releases on GitHub).
:::

## When to reach for giskard-llm

- **You want one code path** for `openai/…`, `google/…`, `anthropic/…`, Azure OpenAI / Azure AI Foundry, **without** going through LiteLLM.
- **You need programmable configuration** (`LLMClient`) for CI, multi-tenant setups, or tests that swap providers cleanly (`reset()`, named aliases).

:::note[Giskard Checks and LLMs]
[Giskard Checks](/oss/checks/installation) document configuring an **LLM judge** via LiteLLM-compatible environment variables—that path is geared toward evaluation workloads. **giskard-llm** is a separate routing layer aligned with native SDK usage; use whichever fits your integration.
:::
